Semantic tags in HTML

Inline text semantics

Semantic tags used inline are not supposed to change how the text looks; instead, they describe the content of the text.

tag tag name description
<strong> strong importance The tag indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
<em> emphasis The tag marks text that has stress emphasis. The element can be nested, with each level of nesting indicating a greater degree of emphasis.
<blockquote> extended quotation The tag indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation. A URL for the source of the quotation may be given using the cite attribute, while a text representation of the source can be given using the cite element.
<q> quotation The tag is used for short quotations. Most modern browsers implement this by surrounding the text in quotation marks.
<abbr> abbreviation The tag represents an abbreviation or acronym; the optional title attribute can provide an expansion or description for the abbreviation.
<cite> cited work The tag is used to describe a reference to a cited creative work, and must include the title of that work. The reference may be in an abbreviated form according to context-appropriate conventions related to citation metadata.
<dfn> definition The tag is used to indicate the term being defined within the context of a definition phrase or sentence. The p element, the dt/dd pairing, or the section element which is the nearest ancestor of the tag is considered to be the definition of the term.
<ins> inserted text The tag represents a range of text that has been added to a document. You can use the del element to similarly represent a range of text that has been deleted from the document.
<del> deleted text The tag represents a range of text that has been deleted from a document. This can be used when rendering "track changes" or source code diff information
<s> strikethrough The tag renders text with a strikethrough, or a line through it. Use the tag to mark things that are no longer relevant or no longer accurate.

Content sectioning

These semantics tags mark the meaning of text sections.

tag tag name description
<address> address The element element indicates that the enclosed HTML provides contact information for a person or people, or for an organization.
<article> article The element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). Examples include: a forum post, a magazine or newspaper article, or a blog entry, a product card, a user-submitted comment, an interactive widget or gadget, or any other independent item of content.
<aside> aside information The element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.
<footer> footer The element represents a footer for its nearest sectioning content or sectioning root element.
<header> header The element represents introductory content, typically a group of introductory or navigational aids.
<main> main content The element represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
<nav> navigation The element represents a section of a page whose purpose is to provide navigation links
<section> section The element represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it.
